ItalianDataA1 processing rules

Current repo model
- Lesson files live only in `lessons/`.
- Each lesson JSON contains Italian source text plus English and Russian localized fields.
- Do not create separate `en/` and `ru/` lesson trees.

Source location
- New source PDFs arrive in `process/`.
- Processed PDFs stay in `IT/`.

Output files
- Create or update one merged lesson file:
  `lessons/a3l3.json`
- File naming rule:
  `A1 Online L3-3.pdf` -> `lessons/a3l3.json`
- Lesson id rule inside JSON:
  `A1-L3-3`

Schema to keep
- Every lesson JSON follows the same top-level structure:
  `lesson_info`
  `language_rules`
  `flashcards`
  `classroom_instructions`
  `dialogues`
  `verb_tables`
- `lesson_info` contains:
  `id`, `title`, `level`, `objectives`
- `objectives` must use:
  `it`, `en`, `ru`
- `language_rules` use localized nested fields:
  `category`, `rule`, `description`
- `flashcards` use:
  `term`, `en`, `ru`, `type`
- `classroom_instructions` use:
  `it`, `en`, `ru`
- `dialogues` use:
  `id`, `title`, `register`, `context`, `turns`, optional `practice_prompts`
- `verb_tables` use:
  `infinitive`, `translation`, `forms`, `pronoun_labels`

How to read the PDFs
- These slide PDFs are presentation exports, not clean textbook pages.
- Inspect enough slides to identify:
  title/unit,
  objectives,
  grammar focus,
  vocabulary focus,
  classroom prompts,
  short dialogue material,
  verb drill pages.
- Do not transcribe every slide.

How to build the lesson JSON
- Preserve Italian in all `it` fields.
- Keep lesson title in Italian.
- Fill English and Russian in sibling locale fields.
- Distill the lesson into practical beginner content.
- Build flashcards from actual lesson vocabulary, not random additions.
- Keep examples short and representative.

Verb rules
- Do not infer verbs only from old JSON.
- Use `IT/*.pdf` as the source of truth.
- See `VERB_EXTRACTION_RULES.md` for the verb extraction rules.
- Rebuild verb tables with:
  `python3 extract_verb_tables.py`

Library rules
- `library.json` is generated, not hand-maintained.
- Rebuild it with:
  `node build_library_json.js`

Index rules
- After lesson changes, update `index.json`.
- Each lesson entry must include:
  `id`, `title`, `level`, `objectives`, `file`,
  `flashcard_count`, `language_rule_count`, `dialogue_count`, `verb_table_count`
- Counts must match the real lesson file.
- Update `generated_at` when content changes materially.

Quality rules
- Keep JSON valid and two-space indented.
- Avoid stale split-locale assumptions.
- Do not use old fields like `translation_en` or `translation_ru` in `verb_tables`.
- Prefer clean beginner phrasing over exhaustive grammar theory.
